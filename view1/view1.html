AngularJS RouteRage

An AngularJS directive which allows you to directly use $routeParams variables in your view
Version: 1.0.0

Result:

Your $routeParams says: [[param1]] [[param2]]
  View on Github

Features

  • Use $routeParams variables in your view directly. No more copying $routeParams to $scope.
  • Using [[ ... ]] brackets to differentiate it from $scope variables.
  • Trigger it once on an element and use it anywhere within the element.
  • Allows you to use simple expressions with your $routeParams, for example: [[productId + 1]].
  • Auto clean up unused [[ ... ]] variables, except on errors.
  • Pure AngularJS. No extra dependency.

Take note that this will only work if the expression is $eval()-able. For example, 10 * 6 will return 60, but "lorem ipsum" + 5 will return undefined (Or worse - I'm not responsible for that).
If it's $eval()-able but still doesn't work.. well maybe your expression is too complicated, sorry :)

Dependency

AngularJS v1.2.1. Other versions may or may not work. Please test first.
